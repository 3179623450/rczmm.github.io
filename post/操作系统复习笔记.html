操作系统介绍

操作系统的历史

第一个操作系统是在 1950 年代和 1960 年代开发的。这些早期的操作系统非常简单，专为大型计算机设计。

在 20 世纪 70 年代，操作系统开始变得更加复杂。这些新操作系统专为个人计算机设计，能够同时支持多个用户和多个程序。

在 20 世纪 80 年代，操作系统变得更加强大和用户友好。这些新操作系统能够支持图形用户界面 (GUI)，并且能够在各种硬件平台上运行。

如今，操作系统对于计算机的运行至关重要。它们为用户提供了运行应用程序、管理文件和访问硬件设备的平台。

操作系统的功能

操作系统执行各种功能，包括：

  • 管理硬件资源：操作系统管理计算机的硬件资源，例如 CPU、内存和 I/O 设备。
  • 提供用户界面：操作系统提供用户界面，允许用户与计算机进行交互。
  • 管理文件：操作系统管理文件，文件是存储在计算机硬盘驱动器上的数据集合。
  • 提供安全性：操作系统提供安全功能来保护用户免受恶意软件和其他威胁。
  • 操作系统的组成部分

操作系统由多种组件组成，包括：

  • 内核：内核是操作系统的核心。它负责管理计算机的硬件资源，并为其他软件提供运行平台。
  • 用户界面：用户界面是用户与操作系统交互的方式。它包括桌面、菜单和图标等元素。
  • 文件系统：文件系统是一种在计算机硬盘驱动器上组织文件的方式。它允许用户存储、检索和管理文件。
  • 设备驱动程序：设备驱动程序是允许操作系统与打印机、键盘和鼠标等硬件设备进行通信的软件。
  • 服务：服务是在后台运行并为操作系统和其他程序提供基本服务的程序。
  • 安全性：操作系统提供安全功能以保护用户免受恶意软件和其他威胁。

进程描述与控制

进程

进程是当前正在运行的程序。

它是一个执行单元，由一个程序计数器、一组寄存器和一个堆栈组成。

程序计数器指向进程将执行的下一条指令。寄存器存储变量的值，堆栈存储调用堆栈。

进程创建

当用户启动一个程序时，操作系统会为该程序创建一个进程。该进程是使用内存中程序代码和数据的副本创建的。该进程也被赋予一个唯一的标识符和一个优先级。

进程调度

操作系统调度进程的执行。调度器决定接下来运行哪个进程。调度程序可以使用多种因素来做出决定，例如进程的优先级、状态以及等待运行的时间。

进程通信

操作系统提供了多种允许进程相互通信的IPC机制。这些机制包括共享内存、管道和套接字。

  • 共享内存

共享内存是一种技术，其中两个或多个进程共享一个公共内存块。这允许进程轻松地彼此共享数据。

  • 管道

管道是一种单向通信通道，允许两个进程相互通信。管道通常用于进程之间的短期通信。

  • 套接字

套接字是一种双向通信通道，允许两个进程相互通信。套接字通常用于进程之间的长期通信。

进程终止

当一个进程终止时，操作系统回收它的资源。回收的资源包括内存、CPU 时间和 I/O 资源。

操作系统管理进程以确保它们高效运行并且不会相互干扰。操作系统使用多种技术来管理进程，例如进程创建、调度和终止。

以下是操作系统在管理进程时面临的一些挑战：

  • 并发

多个进程可以同时运行。操作系统必须确保这些进程不会相互干扰。

  • 僵局

死锁是两个或多个进程互相等待完成的情况。操作系统必须防止死锁的发生。

  • 饥饿

饥饿是一种进程无法运行的情况，因为它总是被其他进程抢占。操作系统必须防止发生饥饿。

操作系统使用多种技术来克服这些挑战，例如：

  • 调度

操作系统使用调度程序来决定接下来运行哪个进程。调度程序可以使用多种因素来做出决定，例如进程的优先级、状态以及等待运行的时间。

  • 同步化

操作系统使用同步技术来确保进程不会相互干扰。同步技术包括互斥、信号量和监视器。

  • 死锁检测和预防

操作系统可以检测死锁并防止它们发生。死锁检测和预防技术包括资源分配图和银行家算法。

  • 避免饥饿

操作系统可以通过使用优先级继承和老化等多种技术来避免饥饿。

处理器调度和死锁

处理器调度

处理器调度是决定下一个进程将在 CPU 上运行的过程。操作系统使用调度程序来做出此决定。调度程序可以使用多种因素来做出决定，例如进程的优先级、状态以及等待运行的时间。

抢占式调度

在抢占式调度中，操作系统可以抢占正在运行的进程并将 CPU 交给另一个进程。这是在正在运行的进程用完其时间片或更高优先级的进程准备好运行时完成的。

非抢占式调度

在非抢占式调度中，进程只有在主动放弃CPU时才能被抢占。这是在进程调用系统调用或进入 I/O 等待

死锁是两个或多个进程互相等待完成的情况。当两个进程各自持有另一个进程需要的资源时，就会发生这种情况。可以使用多种技术来防止死锁，例如资源分配图和银行家算法。

操作系统使用多种技术来调度 CPU 上进程的执行。这些技术允许操作系统确保进程获得公平份额的 CPU 时间，并且没有进程会饿死 CPU 时间。

进程同步

  • 互斥

互斥是一种确保一次只有一个进程可以访问共享资源的技术。这是通过使用锁来完成的。当进程想要访问共享资源时，它必须获得锁。一旦进程获得了锁，它就可以访问共享资源。当进程使用完共享资源时，它必须释放锁。

  • 信号量

信号量是一种同步原语，可用于控制对共享资源的访问。信号量是一个可以保存整数值的变量。信号量的值可以是0或1。当信号量的值为0时，表示共享资源不可用。当信号量的值为1时，表示共享资源可用。

  • 监视器

监视器是一种同步结构，结合了互斥和信号量的特性。监视器是包含共享资源和锁的数据结构。当一个进程想要访问共享资源时，它必须获得锁。一旦进程获得了锁，它就可以访问共享资源。当进程使用完共享资源时，它必须释放锁。

操作系统使用多种技术来确保进程不会干扰彼此的数据。这些技术，如互斥、信号量和监视器，允许操作系统确保进程以安全有序的方式访问共享资源。

以下是操作系统在确保进程同步方面面临的一些挑战：

  • 竞争条件

竞争条件是两个或多个进程试图同时访问同一个共享资源的情况。这可能会导致数据损坏。

  • 僵局

死锁是两个或多个进程互相等待完成的情况。当两个进程各自持有另一个进程需要的资源时，就会发生这种情况。

  • 饥饿

饥饿是一种进程无法访问共享资源的情况，因为它总是被其他进程抢占。

操作系统使用多种技术来克服这些挑战，例如：

  • 互斥

互斥确保一次只有一个进程可以访问共享资源。这可以防止竞争条件。

  • 信号量

信号量用于控制对共享资源的访问。它们可以防止死锁和饥饿。

  • 监视器

监视器是互斥和信号量的组合。它们提供了一种更强大的方式来控制对共享资源的访问。

内存管理

  • 内存分配

内存分配就是给进程分配内存的过程。操作系统必须确保每个进程都有足够的内存来运行，但没有进程使用超过其需要的内存。

  • 分页

分页是一种内存管理技术，它允许操作系统使用比系统中实际存在的内存更多的内存。当进程需要访问当前不在内存中的页面时，操作系统会将页面换出到磁盘并调入另一个页面。

  • 分段

分段是一种内存管理技术，允许操作系统将内存划分为逻辑段。可以为每个段分配其自己的权限，这允许操作系统保护进程免受彼此的影响。

操作系统使用多种技术来管理内存，例如内存分配、分页和分段。这些技术允许操作系统确保进程有足够的内存来运行，没有进程使用超过其需要的内存，并且进程相互保护。

以下是操作系统在管理内存方面面临的一些挑战：

  • 内存碎片

内存碎片是指存在小块空闲内存的情况，这些内存块太小而无法被任何进程使用。当进程分配内存然后释放时，可能会发生这种情况。

  • 内存保护

操作系统必须确保进程不访问它们无权访问的内存。这是通过使用内存保护机制（例如分段和分页）来完成的。

  • 内存开销

操作系统将一些内存用于自己的目的，例如存储进程表和内存映射。此内存称为内存开销。

操作系统使用多种技术来克服这些挑战，例如：

  • 内存压缩

内存压缩是一种可用于减少内存碎片的技术。内存压缩将空闲内存块移动到一起，以便进程可以使用它们。

  • 内存保护

操作系统使用内存保护机制来确保进程不会访问它们无权访问的内存。

  • 减少内存开销

操作系统可以通过使用虚拟内存和分页等技术来减少内存开销。虚拟内存允许操作系统使用比系统中实际存在的内存更多的内存。分页允许操作系统在不使用时将页面换出到磁盘。

虚拟存储

  • 虚拟内存

虚拟内存是一种内存管理技术，它允许每个进程拥有一个大于系统物理内存的虚拟地址空间。这是通过将虚拟地址空间划分为页面，然后只存储当前在物理内存中使用的页面来实现的。当前未使用的页面存储在磁盘上。

  • 分页

分页是一种用于实现虚拟内存的技术。当进程需要访问当前不在物理内存中的页面时，操作系统会将页面换出到磁盘并调入另一个页面。

  • 分割

分段是用于实现虚拟内存的另一种技术。分段将虚拟地址空间划分为多个段，然后每个段都可以分配自己的权限。这允许操作系统保护进程彼此。

  • 地址翻译

地址转换是将虚拟地址转换为物理地址的过程。这是由操作系统使用转换后备缓冲区 (TLB) 完成的。TLB 是一个小表，用于存储虚拟地址和物理地址之间的映射关系。

虚拟内存是现代操作系统的一个重要特性。它允许进程运行，即使它们大于系统的物理内存。它还允许操作系统相互保护进程。

以下是虚拟内存的一些好处：

  • 内存使用量增加

虚拟内存允许进程使用比系统中实际存在的内存更多的内存。这是因为操作系统可以存储当前未在磁盘上使用的页面。

  • 提高性能

虚拟内存可以通过允许操作系统在不使用页面时将页面换出到磁盘来提高性能。这为其他进程释放了物理内存，从而可以提高系统的整体性能。

  • 提高安全性

虚拟内存可以通过允许操作系统相互保护进程来提高安全性。这是因为每个进程都有自己的虚拟地址空间，操作系统可以控制哪些进程可以访问哪些页面。

IO系统

  • 输入/输出 (I/O)

I/O 是在计算机和外部设备之间传输数据的过程。I/O 设备是允许用户与计算机进行交互的硬件设备，例如键盘、鼠标和打印机。

  • 设备驱动程序

设备驱动程序是允许操作系统与 I/O 设备通信的软件程序。设备驱动程序负责将操作系统的指令翻译成 I/O 设备可以理解的命令。

  • 缓冲

缓冲是一种用于提高 I/O 操作性能的技术。当进程从 I/O 设备请求数据时，操作系统首先将数据复制到缓冲区中。缓冲区是位于计算机主内存中的一小块内存。一旦数据在缓冲区中，操作系统就可以将数据传输到进程，而不必等待 I/O 设备完成数据传输。

  • 缓存

缓存是一种通过将频繁访问的数据存储在内存中来提高 I/O 操作性能的技术。当进程从 I/O 设备请求数据时，操作系统将首先检查数据是否已经在缓存中。如果数据在高速缓存中，操作系统就可以将数据传输给进程，而不必等待 I/O 设备完成数据传输。

文件管理

  • 文件

文件是存储在计算机硬盘上的数据集合。文件可以用来存储任何东西，例如文档、图像、音乐和视频。

  • 文件系统

文件系统是一种在计算机硬盘驱动器上组织文件的方式。文件系统将硬盘分为若干个目录，每个目录可以包含若干个文件。

  • 文件管理

文件管理是在计算机硬盘驱动器上创建、删除和访问文件的过程。文件管理通常使用文件管理器完成，文件管理器是允许用户与文件系统交互的软件程序。

  • 文件创建

文件创建是在计算机硬盘驱动器上创建新文件的过程。要创建新文件，用户通常使用文件管理器打开一个新目录，然后输入文件名。

  • 文件删除

文件删除是从计算机硬盘驱动器中删除文件的过程。要删除文件，用户通常使用文件管理器选择文件，然后按删除键。

  • 文件存取

文件访问是打开计算机硬盘驱动器上的文件的过程。为了访问文件，用户通常使用文件管理器来选择文件然后按下打开键。

文件管理是使用计算机的重要部分。它允许用户组织他们的文件并快速轻松地访问它们。

以下是操作系统在管理文件时面临的一些挑战：

  • 文件碎片

文件碎片是文件存储在硬盘驱动器上的非连续块中的情况。当文件被删除然后重新创建，或者文件被四处移动时，可能会发生这种情况。文件碎片会减慢文件访问速度。

  • 文件安全

操作系统必须确保文件不会受到未经授权的访问。这是通过使用文件权限来完成的，文件权限控制谁可以访问文件以及他们可以对文件做什么。

  • 文件备份

操作系统必须确保备份文件以防数据丢失。这是通过使用文件备份工具完成的，该工具将文件复制到安全位置。

操作系统使用多种技术来克服这些挑战，例如：

  • 碎片整理

碎片整理是一个可用于减少文件碎片的过程。碎片整理移动文件，以便它们存储在硬盘驱动器上的连续块中。

  • 文件权限

文件权限用于控制谁可以访问文件以及他们可以对文件做什么。文件权限可以由用户或操作系统设置。

  • 文件备份

文件备份是将文件复制到安全位置的过程。文件备份可以手动或自动完成。

磁盘存储管理

  • 文件分配

文件分配是将磁盘上的空间分配给文件的过程。创建文件时，操作系统会在磁盘上为该文件分配一块空间。

  • 碎片化

碎片是文件存储在磁盘上的非连续块中的情况。当文件被删除然后重新创建，或者文件被四处移动时，可能会发生这种情况。碎片会减慢文件访问速度。

  • 袭击

RAID（独立磁盘冗余阵列）是一种可用于提高磁盘存储性能和可靠性的技术。RAID 的工作原理是将多个磁盘组合成一个逻辑磁盘。

文件分配是磁盘存储管理的重要组成部分。它确保文件以高效的方式存储并且可以快速访问。

碎片会减慢文件访问速度。它还会使删除文件变得困难。碎片整理是一个可用于减少碎片的过程。

RAID 是一种可用于提高磁盘存储性能和可靠性的技术。RAID 的工作原理是将多个磁盘组合成一个逻辑磁盘。这可以通过将负载分散到多个磁盘来提高性能。它还可以通过在其中一个磁盘发生故障时提供备份来提高可靠性。

以下是操作系统在管理磁盘存储方面面临的一些挑战：

  • 磁盘空间管理

操作系统必须确保有效地使用磁盘空间。这是通过使用文件分配技术完成的，例如连续分配和链接分配。

  • 磁盘碎片

磁盘碎片会减慢文件访问速度。操作系统可以使用碎片整理工具来减少碎片。

  • 磁盘故障

磁盘可能会发生故障。操作系统必须确保在磁盘发生故障时数据不会丢失。这是通过使用 RAID 技术完成的。

操作系统使用多种技术来克服这些挑战，例如：

  • 文件分配技术

操作系统使用文件分配技术来确保有效地使用磁盘空间。连续分配将文件存储在磁盘上的连续块中。链接分配将文件存储在磁盘上的非连续块中。

  • 碎片整理工具

操作系统使用碎片整理工具来减少磁盘碎片。碎片整理工具移动文件，以便它们存储在磁盘上的连续块中。

  • RAID技术

操作系统使用 RAID 技术来提高磁盘存储的性能和可靠性。RAID 的工作原理是将多个磁盘组合成一个逻辑磁盘。这可以通过将负载分散到多个磁盘来提高性能。它还可以通过在其中一个磁盘发生故障时提供备份来提高可靠性。
